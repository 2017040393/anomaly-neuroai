# NeuroAI 工业异常检测路线图与第二阶段完成清单

这份文档用于记录当前项目的长期路线、阶段目标，以及第二阶段的完成标准。

当前项目定位：

> 以工业视觉异常检测为切入点，先完成真实数据集上的主流 baseline 学习与实践，再逐步探索轻量化、实时性、在线适应和后续 NeuroAI 增强，并在条件成熟后考虑与机器人相关体系结合。

---

# 一、项目总路线

整体分为三个阶段：

1. 第一阶段：工程骨架与最小可运行流程
2. 第二阶段：正式入门工业视觉异常检测，接入真实数据与主流 baseline
3. 第三阶段：差异化探索与 NeuroAI 增强

---

## 第一阶段：工程骨架与最小可运行流程

### 阶段目标

这一阶段的核心不是追求真实工业效果，而是先把整个工程基础打稳。

要完成的事情包括：

- 使用 `uv` 建立项目环境
- 使用 `pyproject.toml` 管理依赖
- 搭建 `src/` 风格项目结构
- 打通训练、评估、demo 的最小流程
- 支持本地小规模实验
- 支持后续迁移到云端训练
- 使用 synthetic dataset 跑通基础逻辑
- 支持最小 `mlp` / `snn_mlp` 模型切换
- 能保存 checkpoint 和 `metrics.csv`

### 第一阶段完成标志

如果满足以下条件，则认为第一阶段完成：

- 项目目录结构已稳定
- `uv` 工作流可正常使用
- 配置文件可通过 `--config` 加载
- 最小训练脚本可运行
- 最小评估脚本可运行
- demo 可运行
- synthetic dataset 上能得到正常输出
- 本地小显卡可用于调试
- 后续新增真实数据集时无需推翻整体结构

### 第一阶段的意义

第一阶段的意义不是“已经进入工业异常检测”，而是：

> 已经具备一个可扩展的工程地基，可以在其上继续做真实工业视觉异常检测。

---

## 第二阶段：正式入门工业视觉异常检测

### 阶段目标

第二阶段是当前最重要的阶段。

这一阶段的核心目标是：

> 正式进入工业视觉异常检测方向，接入真实数据集，跑通主流 baseline，理解任务定义、输出形式、评价指标和工程组织方式。

第二阶段不是创新阶段，而是：

- 学这个方向
- 亲手做这个方向
- 理解主流方法
- 跑出真实结果
- 形成第一版像样的工业异常检测作品

### 第二阶段需要做的事

#### 1. 接入真实工业视觉异常检测数据集

优先支持：

- MVTec AD

后续再考虑：

- VisA
- MVTec LOCO AD
- MVTec AD 2

#### 2. 理解任务设定

需要真正搞清楚：

- 什么是正常样本
- 什么是异常样本
- train / test 分别怎么用
- 图像级异常检测是什么
- 像素级异常定位是什么
- anomaly score 是什么
- anomaly map 是什么
- heatmap / overlay 的意义是什么

#### 3. 跑通主流 baseline

优先顺序建议：

1. PatchCore
2. PaDiM
3. EfficientAD 或 DRAEM（后续）
4. 更复杂的方法（更后续）

第二阶段至少应完成：

- PatchCore 跑通
- PaDiM 跑通
- 最好能做一次两者对比

#### 4. 增加真实 anomaly detection 流程

需要补充：

- MVTec dataset 读取
- anomaly transforms
- anomaly training / inference 入口
- anomaly map 保存
- overlay 图保存
- image-level score 导出
- 推理延迟测试脚本
- 简单 Gradio demo

#### 5. 形成第一版可展示作品

这一阶段的目标不是论文创新，而是做出一个别人能理解的作品：

- 输入真实工业图像
- 输出异常分数
- 输出异常热图或 overlay
- 能展示正常/异常差异
- 能保存结果图
- 能简单比较不同 baseline

---

## 第三阶段：差异化探索与 NeuroAI 增强

### 阶段目标

第三阶段不是当前必须马上进入的阶段，而是在第二阶段真正跑顺之后，再逐步开始。

第三阶段的核心问题是：

> 当我已经会做工业视觉异常检测，并能跑主流 baseline 以后，我自己的差异化价值在哪里？

### 第三阶段可能的方向

可考虑的方向包括：

- 轻量化
- 更低延迟
- 更适合边缘部署
- 在线适应
- temporal memory
- 稀疏表示
- 小型 SNN 对照
- 对结构异常更敏感
- 更适合机器人视觉巡检场景
- 后续与机器人体系结合

### 第三阶段的意义

这一阶段才开始逐步进入：

- 你自己的改进
- 你的工程主张
- NeuroAI 风格增强
- 与机器人或工业部署场景结合

第三阶段不应该先于第二阶段。

---

# 二、当前最推荐的技术主线

结合已有基础（语义分割背景）与当前条件，当前最推荐路线为：

> 工业视觉异常检测 + 可视化定位 + 主流 baseline 学习与实践

原因：

- 与已有视觉基础衔接自然
- 易于做结果展示
- 易于做 anomaly map / heatmap
- 更容易形成“像作品”的 demo
- 后续更容易扩展到机器人视觉巡检与工业感知模块

---

# 三、当前推荐的学习与实现顺序

建议按以下顺序推进，而不要一开始到处发散。

## 第一层：先理解数据与任务

优先内容：

- MVTec AD 数据结构
- 正常/异常样本组织方式
- ground truth mask 的形式
- image-level vs pixel-level 任务
- AUROC 等常用评价指标
- anomaly map 的可视化意义

## 第二层：先跑通主流经典 baseline

优先顺序：

1. PatchCore
2. PaDiM

目标：

- 跑通真实数据
- 得到 anomaly score
- 得到 anomaly map
- 保存 overlay
- 做一次 baseline 对比

## 第三层：做成作品而不是只停在代码

目标：

- 可以导出结果图
- 可以展示几张典型成功/失败样例
- 可以做一个 demo
- 可以记录延迟、显存、速度

## 第四层：后续再加扩展方法

在第二阶段基本完成后，再考虑：

- EfficientAD
- DRAEM
- Reverse Distillation
- WinCLIP / foundation model 路线

## 第五层：最后再做你的 NeuroAI 增强

当经典方法已经跑顺之后，再考虑：

- temporal smoothing
- online adaptation
- 稀疏表示
- 小型 SNN 路线
- 更轻量或更快的部署方案

---

# 四、第二阶段的明确边界

为了防止自己在第二阶段过度发散，这里明确写清楚：

## 第二阶段主要做什么

- 正式入门工业视觉异常检测
- 接真实数据集
- 跑主流 baseline
- 形成 anomaly detection 的完整工程流程
- 做结果图与 demo
- 逐渐建立对该方向的直觉

## 第二阶段不强求什么

- 不强求创新
- 不强求立刻做 NeuroAI 差异化
- 不强求立刻和机器人系统打通
- 不强求一上来追最新最重的方法
- 不强求立刻做创业化判断

第二阶段最重要的一句话是：

> 先把工业视觉异常检测这门手艺学会。

如果这些条件大部分满足，则可逐步进入第三阶段，开始探索：

# 七、当前阶段不建议做的事

为了避免路线发散，当前阶段尽量不要做：

- 一上来追大量最新 foundation model 论文
- 一上来把系统全改成 SNN
- 一上来就碰太复杂的数据集和任务
- 一上来试图把创业、创新、NeuroAI、机器人全部合并
- 一次性做太大的系统
- 过早为“未来产品化”写太重的基础设施

当前最重要的是：

> 先在真实工业视觉异常检测上把 baseline 学会、跑顺、做成作品。

---

# 八、当前阶段的简洁行动主线

可以把接下来的主线压缩成下面几步：

1. 接 MVTec AD
2. 跑 PatchCore
3. 跑 PaDiM
4. 保存 anomaly map 与 overlay
5. 做一个最小 demo

# 九、当前阶段的核心提醒

当前项目还不需要急着回答：

- 我的最终创新是什么？
- 我的最终创业产品是什么？
- 我的最终 NeuroAI 体系是什么？

当前更需要回答的是：

- 我是否已经真正会做工业视觉异常检测？
- 我是否已经跑通真实 baseline？
- 我是否已经有一个能展示的作品？

只要这三个问题开始有肯定答案，后面的路就会清楚很多。

# 十、一句话总结

当前最重要的目标不是“立刻做出差异化创新”，而是：

> 先完成工业视觉异常检测的正式入门，并在真实数据集上跑通主流 baseline，做出一个能展示、能比较、能继续扩展的作品。

当这一步完成之后，再进入第三阶段，去做你的轻量化、在线适应、NeuroAI 增强，以及未来可能的机器人结合路线。
